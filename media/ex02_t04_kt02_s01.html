Exercise 02 — Conflicting Signals Notes

Each stage's benign explanation (what an analyst saw in real time) paired with its malicious tell (the giveaway visible only in hindsight or with cross-layer context).

Stage Benign Explanation (real-time read) Malicious Tell (cross-layer giveaway)
1 — Initial access (Identity) Medium-severity anonymousIpAddress detection — VPN and privacy-tool use is common; easy to triage as a remote worker on a VPN, not an attacker. The flagged IP (102.89.42.17) is a known Tor exit node / anonymous proxy. Combined with the subsequent grounding-source upload minutes later by the same identity, this is initial-access pivot — not a VPN.
2 — Poisoning (grounding source) A policy document uploaded to a knowledge base looks exactly like routine business — vendors send policy updates constantly; nothing about the upload action itself is malicious-looking. The upload occurred immediately after the stolen-session sign-in (Stage 1) and targeted the Refund Agent's grounding container specifically. The document contains a hidden white-text prompt-injection payload.
3 — Reasoning probe (AI) ★ Defender for Cloud raised a medium-severity AI alert — but as one item among ~25 decoys, "the agent is busy / likely a false positive" is the easy read. Nothing ties it to identity or infra activity. The alert fired on the same Azure OpenAI resource (oai-soclab-v02) whose grounding source was poisoned 18 minutes earlier by the same compromised identity. Cross-layer, it is not ambiguous — but no rule made that join.
4 — Lateral movement (Infrastructure) A service principal named sp-refund-agent-inference requesting inference compute is literally its job description — anomalous volume is the only tell, and only meaningful against a known baseline. The token requests arrived immediately after the AI probing stage and targeted production infrastructure (rg-zava-ai-prod) outside the principal's normal scope. Paired with Stages 1–3, this is lateral movement, not routine inference.
5 — Execution (Endpoint) A script named update_check.ps1 reads as routine IT maintenance — patch-check scripts run constantly on managed endpoints. The script runs an encoded PowerShell command and beacons outbound — encoded commands also appear in legitimate management tooling, but the outbound indicator and timing (following cloud-layer activity) make this execution, not maintenance.

Meta-point: Each signal had an innocent explanation that was more probable than "coordinated multi-stage attack" when viewed in isolation. The conflict is not signal vs. noise — it is plausible-benign vs. plausible-malicious, repeated five times across five consoles, with no view that aggregates the priors. That is precisely what cross-layer, machine-speed correlation resolves.
